# 1. Executive Summary (สรุปภาพรวม)

## Goal (เป้าหมายของโปรเจค)
พัฒนาระบบสั่งอาหารสำหรับร้านพิซซ่าที่จำลองรูปแบบการบริการจัดการคิวและออเดอร์สไตล์ฟาสต์ฟู้ด (KFC Model)  
เพื่อลดความผิดพลาดในการรับออเดอร์ และเพิ่มความรวดเร็วในการให้บริการลูกค้าตั้งแต่ขั้นตอนการสั่งจนถึงการรับอาหาร

## Platform (แพลตฟอร์มที่ใช้งาน)
- **Staff Facing:** Web Application สำหรับพนักงานหน้าร้าน (Cashier/Counter)

## Key Value (คุณค่าหลักที่ส่งมอบ)
- **Operational Efficiency:**  
  ครัวเห็นออเดอร์ทันทีที่ลูกค้าสั่งเสร็จ ช่วยลดเวลาในการส่งต่อข้อมูล  
  และจัดการออเดอร์ได้ตามลำดับอย่างเป็นระบบ รวมถึงลด Human Error ในการจดรายการ


# 2. Scope

## In-Scope (สิ่งที่ต้องทำ - Core Functions)

### ระบบพนักงานรับออเดอร์ / แคชเชียร์ (POS/Cashier System)
- ดูเมนูอาหาร (มีหลายเมนูและรูปแบบให้เลือก: พิซซ่า, ของทานเล่น, เครื่องดื่ม, เซ็ตคอมโบ)
- ระบบตะกร้าสินค้า
- ระบบคำนวณราคารวม
- ออกหมายเลขคิว (Order Number) เมื่อทำการยืนยันออเดอร์
- ดูรายการออเดอร์ที่เข้ามาและกดรับชำระเงิน (จำลองการจ่ายเงิน)
- สั่งพิมพ์ใบเสร็จและใบออเดอร์ส่งเข้าครัว


## Out-of-Scope (สิ่งที่ไม่ทำในเฟสนี้ เพื่อควบคุมเวลาพัฒนา)
- ระบบ Delivery และการติดตาม GPS ของ Rider  
  (โฟกัสเฉพาะ Dine-in และ Takeaway ก่อน)
- การเชื่อมต่อกับ Payment Gateway ตัดบัตรเครดิตจริง  
  (ใช้ระบบ Mock-up หรือยืนยันสลิปด้วยคนแทน)
- ระบบสมาชิก (Membership), การสะสมแต้ม (Loyalty Program),  
  และคูปองส่วนลดแบบซับซ้อน
- ระบบจัดการสต๊อกวัตถุดิบหลังบ้านแบบละเอียด  
  (Inventory & Ingredient Tracking)

### ระบบจัดการครัว (Kitchen Display System - KDS)
- แสดงรายการอาหารที่ต้องทำตามลำดับคิว
- พนักงานครัวสามารถกดอัปเดตสถานะออเดอร์ได้  
  (เช่น กำลังทำ → เสร็จแล้ว)

### ระบบแสดงสถานะคิว (Queue Monitor)
- หน้าจอแสดงหมายเลขคิว โดยแบ่งเป็น 2 สถานะหลัก:
  - กำลังเตรียม (Preparing)
  - เชิญรับอาหาร (Please Collect)
